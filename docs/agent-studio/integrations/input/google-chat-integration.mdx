## Delete Message

Delete a message in `Google Chat`.

### Settings

#### Client ID & Client Secret

For a visual step-by-step guide, watch this [video](#google-oauth-setup).

For seamless integration of `Google Chat` with `SmythOS`, obtaining secure credentials such as a `Client ID` and a `Client Secret` is essential. Below is a step-by-step guide to help you acquire these credentials:

1. **Visit the Google Developers Console**:
   - Navigate to the [Google Developers Console](https://console.developers.google.com/).
   - Create a new project or select an existing project that is appropriate for your needs.

2. **Enable the Google Chat API**:
   - Proceed to the `Library` section within the Developers Console.
   - Search for the `Google Chat API` and enable it for your selected project.

3. **Configure the OAuth Consent Screen**:
   - Go to the `OAuth consent screen` tab.
   - Fill in the required details such as your application's name, email address, and the scopes necessary for your application on the consent screen settings.
   - Save and continue to set up the OAuth consent screen.

4. **Create Credentials**:
   - Navigate to the `Credentials` tab in the console.
   - Click `Create credentials`, then select `OAuth client ID`.
   - Choose `Web application` as the application type.
   - Under `Authorized redirect URIs`, add the following URL: `https://app.smythos.com/oauth/google/callback`
   - Click `Create` to generate your OAuth client ID and client secret.

5. **Retrieve Client ID and Client Secret**:
   - Once the OAuth client ID is configured, the system will provide you with a `Client ID` and a `Client Secret`.
   - Copy and store these credentials securely as they are necessary for configuring the Google Chat integration in SmythOS.

6. **Enter and Authenticate**:
   - Input the acquired `Client ID` and `Client Secret` into the relevant fields in the SmythOS settings for the Google Chat component.
   - Click on the `Authenticate` button to establish a secure connection, allowing you to utilize Google Chat within SmythOS efficiently.

> **Note:** Treat the `Client ID` and `Client Secret` as sensitive data. In `SmythOS`, you can securely manage it using the `Vault` feature. For more details, please refer to [this](https://app.smythos.com/doc/#/vault/keysManagement) section in the documentation.

<Image src="/img/google_chat/screenshot4.png" alt="alt text" /> 

### Inputs

To effectively utilize the component for deleting a `Google Chat` message, provide the following inputs:

- **spaceId (`required<string>`)**: The unique identifier of the `Google Chat` space where the message will be deleted.
- **messageId (`required<string>`)**: The unique identifier of the message to be deleted.

<Image src="/img/google_chat/screenshot5.png" alt="alt text" />

> **Note:** With the flexibility and customization in mind, `SmythOS` allows you to expand the default input list by adding new inputs using the `+` button situated in the `Inputs` section.

### Outputs

After processing the inputs, this feature generates detailed outputs, including:

1. **Response**: Contains the response from the `Google Chat` API, confirming the deletion of the message.
2. **Headers**: Includes HTTP headers that provide insights into the API request-response process, such as status codes and metadata.
