# Agent Orchestration

In **SmythOS**, an **agent** is a packaged workflow that can **reason**, **act**, and **respond** across multiple channels (API, chat, voice, batch).  
Agent orchestration is the practice of **designing, running, and governing** these agents so they meet your product and operational needs.

> Put another way: **Workflows** describe _how_ logic flows, while **agents** wrap that logic with memory, auth, and deployment metadata so you can ship it to users.
